# PROJECT: HW 복원 정책 초안

## 목적

웹 전용 게임이라도 사용자가 기기를 바꾸거나 브라우저 저장소가 삭제될 수 있다. 복원은 로컬 저장이 아니라 동일 계정의 서버 기록을 기준으로 수행한다.

## 복원 대상

| 대상 | 처리 |
| --- | --- |
| 영구 상품 | 자동 복원 |
| 하우징 가구/패키지 | 구매/권한 기록 기준 복원 |
| 블링 잔액 | 서버 지갑 잔액 동기화 |
| 인벤토리 | 서버 저장 기준 동기화 |
| 구매 내역 | 서버 기록 기준 조회 |
| 시즌 권한 | 활성 기간과 권한 상태 기준 복원 |

## 복원 비대상

| 대상 | 이유 |
| --- | --- |
| 로컬 UI 설정 | 기기별 편의 상태 |
| 임시 필터/정렬 | 서버 복원 가치 낮음 |
| 만료된 소모성 혜택 | 기록 보존만 수행 |
| 사용 완료된 소모품 | 구매 기록은 남기되 재지급하지 않음 |

## 자동 복원 흐름

1. 사용자가 로그인한다.
2. 서버가 `userId` 기준 지갑, 저장 데이터, 권한, 구매 기록을 조회한다.
3. 서버가 활성 권한과 잔액을 응답한다.
4. 클라이언트가 로컬 UI 상태를 병합한다.
5. 서버가 `restore_events`에 복원 결과를 기록한다.

## 복원 응답 초안

```ts
type RestoreSummary = {
  userId: string;
  wallet: {
    coin: number;
    freeBling: number;
    paidBling: number;
  };
  restoredEntitlements: Array<{
    entitlementId: string;
    productId: string;
    status: "active";
  }>;
  playerSaveVersion: number;
  restoredAt: string;
};
```

## 복원 실패 처리

복원 중 일부 데이터가 실패하면 클라이언트는 결제와 지급 관련 UI를 제한하고 재시도를 안내한다. 서버는 실패 원인을 `restore_events`에 남긴다.

## 완료 기준

동일 계정으로 로그인하면 서버에 남은 잔액, 구매 상품, 영구 권한이 자동으로 표시되어야 한다.
